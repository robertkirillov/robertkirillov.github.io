Rules to remember

      glia. prefix is reserved       50 reserved (glia.*) max       20 custom attributes max       100 ops per request       ~2 year TTL [3:37 PM]

Custom attribute keys must not start with glia. - that namespace is reserved by the platform. Method replace overwrites the whole custom set; merge only adds/updates the keys you send. Picking replace when you meant merge silently drops attributes set earlier in the session - a frequent support ticket.

Ticket states - active: enqueued, request_sent; intermediate: pending_termination; erroneous: unstaffed, disconnected, visitor_conflict, forbidden, full; terminal: finished, canceled, failed.

      

An operator is suitable when: in one of the ticket's queues, presence is available, no conflict in the last 30s, capacity not exceeded (multi_engagement_max_count), and the media is engageable. When several qualify, ranking breaks ties by fewest active engagements (or explicit operator rank when operator_ranking_enabled). No suitable operator → unstaffed. Messaging tickets carry a 24h SLA and a fallback_queue_id can catch overflow.
